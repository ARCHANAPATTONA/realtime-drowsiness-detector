# Driver Drowsiness Detection and Emergency Alert System

A real-time, software-only prototype that detects driver fatigue using a webcam,
sounds an alarm, and dispatches an emergency SMS if the driver remains unresponsive.

---

## Folder Structure

```
drowsiness_detection_system/
├── main.py                  ← Entry point
├── face_detection.py        ← MediaPipe face mesh wrapper
├── drowsiness_detection.py  ← EAR/MAR logic + state machine
├── alert_system.py          ← Alarm + SMS + location
├── config.json              ← All tuneable parameters
├── requirements.txt
├── generate_alarm.py        ← Utility: creates assets/alarm.wav
└── assets/
    └── alarm.wav            ← Generated by generate_alarm.py
```

---

## Installation

### 1. Prerequisites

- Python 3.9 or higher
- A working webcam
- (Optional) Twilio account for real SMS

### 2. Create a virtual environment (recommended)

```bash
python -m venv venv
# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Generate the alarm sound

```bash
python generate_alarm.py
```

This creates `assets/alarm.wav` using pure Python (no extra dependencies).

### 5. (Optional) Configure Twilio for real SMS

Edit `config.json`:

```json
"alert": {
    "use_mock_sms": false
},
"twilio": {
    "account_sid": "ACxxxxxxxxxxxxxxxx",
    "auth_token":  "your_auth_token",
    "from_number": "+1XXXXXXXXXX",
    "to_number":   "+1XXXXXXXXXX"
}
```

## Running the System

```bash
# Standard run
python main.py

# With a different camera
python main.py --camera 1

# Disable alarm sound (testing)
python main.py --no-alert

# Custom config file
python main.py --config /path/to/config.json
```

### Keyboard controls (while running)

| Key | Action |
|-----|--------|
| `q` | Quit |
| `r` | Reset emergency state (acknowledge alert) |

## How It Works

### Eye Aspect Ratio (EAR)

```
EAR = (||p2-p6|| + ||p3-p5||) / (2 × ||p1-p4||)
```

- Six MediaPipe landmarks per eye form this ratio.
- Open eye ≈ 0.30 · Closed eye ≈ 0.15
- **Threshold**: 0.25 (configurable)

### Mouth Aspect Ratio (MAR)

```
MAR = (A + B + C) / (3 × D)
```

- Eight landmarks; A/B/C are vertical distances, D is horizontal width.
- Relaxed ≈ 0.3 · Yawning ≈ 0.7+
- **Threshold**: 0.60 (configurable)

### State Machine

```
              eyes open                reset
ALERT  ──────────────────────────────────────────────
  │  eyes closed > EAR_THRESHOLD for DROWSY_TIME_SEC
  ▼
DROWSY  ── alarm sounds ── countdown timer starts
  │  still closed for EMERGENCY_TIME_SEC
  ▼
EMERGENCY  ── SMS sent ── press 'r' to reset
```

Yawning is detected in parallel and does not affect the eye-closure timer.

## Threshold Tuning Guide

| Parameter | Default | Effect of increasing | Effect of decreasing |
|-----------|---------|---------------------|---------------------|
| `EAR_THRESHOLD` | 0.25 | More sensitive (triggers easier) | Less sensitive (may miss drowsiness) |
| `MAR_THRESHOLD` | 0.60 | Less yawn-sensitive | Triggers on normal mouth movement |
| `EAR_CONSEC_FRAMES` | 20 | Requires longer closure | Faster alarm trigger |
| `DROWSY_TIME_SECONDS` | 2.5 | More tolerant of slow blinks | Earlier alarm |
| `EMERGENCY_TIME_SECONDS` | 12 | More time before SMS | Faster emergency dispatch |

### Recommended calibration steps

1. Run with `--no-alert` and watch the EAR readout in the HUD.
2. Note your normal open-eye EAR (typically 0.28–0.35).
3. Set `EAR_THRESHOLD` to ~80% of your open-eye value.
4. Similarly, open your mouth wide and note MAR; set `MAR_THRESHOLD` to ~85% of that.

## Performance Notes

- Runs at 25–30 FPS on a mid-range laptop (CPU only).
- `CAP_PROP_BUFFERSIZE=1` minimises camera latency.
- MediaPipe Face Mesh uses CPU by default; GPU acceleration is available
  by setting `model_complexity=1` in `FaceDetector.__init__`.
- For Raspberry Pi / embedded targets, reduce `frame_width`/`frame_height`
  in config to 640×480 and set `model_complexity=0`.
